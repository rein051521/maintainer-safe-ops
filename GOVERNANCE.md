# Governance

## Project model

maintainer-safe-ops is currently maintained by a single primary maintainer.

## Decision making

Changes are accepted based on:

- maintainer workflow value
- security benefit
- test coverage
- low false-positive risk
- documentation quality
- maintainability

## Maintainer responsibilities

The primary maintainer is responsible for:

- reviewing issues and pull requests
- release management
- rule quality
- security policy updates
- roadmap prioritization
- dependency and workflow maintenance

## Becoming a contributor

Contributors can become regular contributors by submitting scoped, tested, and documented pull requests.

## Security-sensitive changes

Changes to GitHub Actions workflows, secret detection logic, SARIF output, release behavior, or dependency configuration require careful review.
