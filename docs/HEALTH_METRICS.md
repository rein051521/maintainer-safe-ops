# Project Health Metrics

This document defines the metrics used to maintain this project honestly.

## Activity

Track:

- commits in the last 90 days
- releases in the last 90 days
- open issues
- closed issues
- open pull requests
- merged pull requests

## Responsiveness

Track:

- time to first response for issues
- time to close issues
- time to review pull requests

## Quality

Track:

- CI pass rate
- test count
- lint status
- CodeQL status
- OpenSSF Scorecard status
- Dependabot alert status

## Security

Track:

- security policy presence
- secret scanning settings
- branch protection settings
- workflow permission minimization
- CodeQL and Scorecard runs

## Honesty rule

Do not claim adoption, usage, downloads, or external contributors unless the evidence exists.
