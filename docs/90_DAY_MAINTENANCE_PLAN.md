# 90-Day Maintenance Plan

## Week 1

- Publish repository
- Confirm CI is green
- Create v0.1.0 release
- Create 4 roadmap issues
- Enable GitHub security settings
- Run CodeQL and OpenSSF Scorecard

## Weeks 2-4

- Merge 2-3 scoped PRs:
  - improve SARIF metadata
  - add allowlist config
  - add Japanese maintainer guide
- Close at least one issue with a linked PR
- Review Dependabot alerts if any

## Month 2

- Add more GitHub Actions workflow rules
- Add pre-commit usage example
- Improve false-positive documentation
- Contribute 1-3 small PRs to existing OSS projects

## Month 3

- Publish v0.2.0
- Write a short project report:
  - what changed
  - what was learned
  - what false positives remain
  - what maintainers can use it for
